# Task Tracker (Java + JDBC + PostgreSQL)

##  Описание проекта

Task Tracker — это консольное приложение для управления задачами, вдохновлённое такими системами как Jira.

Проект реализован с использованием:

* Java (без фреймворков)
* JDBC
* PostgreSQL

Основная цель проекта — отработка:

* работы с базой данных
* написания бизнес-логики
* построения архитектуры (Service + Repository)
* понимания связей между сущностями

---

##  Основной функционал

###  Пользователи

* Регистрация пользователя
* Авторизация по email и паролю
* Проверка уникальности username и email

---

###  Проекты

* Создание проекта
* Назначение владельца проекта
* Добавление пользователей в проект
* Управление ролями (admin / member)

---

###  Участники проекта

* Связь пользователей и проектов (many-to-many)
* Проверка доступа к проекту
* Разграничение прав

---

###  Задачи

* Создание задачи внутри проекта
* Назначение исполнителя
* Обновление задачи (название, описание)
* Удаление (soft delete)
* Получение списка задач

---

###  Статусы задач (Workflow)

* TODO / IN_PROGRESS / DONE
* Ограничение переходов между статусами
* Проверка допустимости смены статуса

---

###  Комментарии

* Добавление комментариев к задачам
* Просмотр комментариев

---

###  История изменений

* Логирование всех изменений задачи:

    * смена статуса
    * смена исполнителя
    * изменение данных
* Хранение:

    * старого значения
    * нового значения
    * пользователя
    * времени изменения

---

###  Model

Содержит только данные:

* User

* Comment

---

### ️ Repository

Отвечает за SQL-запросы:

### ️ Service

##  Структура базы данных

### Users

* id

* password
* created_at

---

### Projects

* id
* name
* description

* created_at

---

### Project Members

---

### Tasks

* description
* status_id

* creator_id
* assignee_id
* project_id
* created_at
* due_date

### Task Statuses

### Comments

### Task History

* id

---

##  Бизнес-правила

* Пользователь должен быть участником проекта для работы с задачами
* Нельзя назначить задачу на пользователя вне проекта
* Только допустимые переходы статусов (workflow)
* Все изменения задачи записываются в историю
* При создании проекта владелец автоматически становится admin

---

##  Авторизация

* Авторизация осуществляется по email + password
* Пользователь считается авторизованным, если найден в БД
* В текущей версии пароль хранится в открытом виде (планируется хэширование)

---

##  Транзакции

Критические операции выполняются в транзакциях:

* создание задачи + запись истории
* изменение статуса + логирование

---

## Как запустить

1. Установить PostgreSQL
2. Создать базу данных
3. Выполнить SQL-скрипты (создание таблиц)
4. Указать настройки подключения в `resources.properties`
5. Запустить `Main.java`
